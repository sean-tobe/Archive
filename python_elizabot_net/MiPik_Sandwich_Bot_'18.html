MiPik Sandwich Bot

        MiPik, whose name is a play on Raspberry “Pi” and how the Sandwich Bot “mimics”  the customer’s sub order back to them as the conversation continues. I drew inspiration from standing in line for Subway at school, and realized the boring employee-customer interaction of moving down the sandwich work-line (size, bread, protein, veggies, toppings). So I decided my implementation of an Eliza conversation bot will utilize Eliza’s straightforward answer-response text format as a template of how conversation with MiPik Subs should unfold.

        MiPik bot scans the server chat and checks the data line by line to see if any messages are directed to |MiPik| and responds to them in order.

If the message contains (order|want|get|have|need|add|select|choose|put), the customer is responding to order more to their custom sub. The following string after the order keyword is stored into a dictionary “orders”, with the key = customer name and entry = their order. If the customer is not in the dictionary, make an entry, else concatenate their message into their dictionary entry for custom sub order.

If the message contains (confirm|done|finish|end|exit|quit), the customer is confirming their order, so MiPik deletes the entry in the dictionary to keep track of current customers.

If there are no entries (no current customers), MiPik will advertise in the chat, to prompt other bots to talk directly to MiPik.

        Once an order is started by keywords (begin|start|go), MiPik begins sandwich-making template. MiPik asks the customer questions from “What size?” to “Which bread, protein, etc..”

MiPik keeps tracks of the customer’s position in the process by also storing the name and an integer into dictionary “substep”, which prompts which of the 5 order responses MiPik should say. For ex, if for customer {“Alice”: 2}, Alice has already started a size and declared bread, so MiPik will ask “What veggies should I add?”.

I did not expect some other bots to reiterate MiPik’s sandwich question back as the corresponding order, so sometimes there’s a customer sub missing the proper nouns. However, I also wanted my bot to fulfill any requests that’s given because it’s purpose is to just be happy making sandwiches. This template could be applied to any topic but I choose sandwiches because of the multiple categories that can contain any arbitrary list, which is easily put together as a string by MiPik!
